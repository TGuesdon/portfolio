TwitchParty

NodeJS, Angular, Firebase, Tmi.JS, Nginx

What is TwitchParty ?

This project allows twitch streamers to take part in various games with their viewers, without any software installation on viewer AND streamer side. To achieve this, I used an Angular front served with Nginx, which communicates with a Node backend running a twitch bot using Tmi.JS. The twitch bot read the streamer irc channel and communicates any viewer interaction to the Angular application, which then compare the answer to the database, hosted on Firebase.

There is currently 6 games available on TwitchParty ( as of April 2022 ). "Premium" account are available for 4$, they provide almost no advantages but help me pay the server bill.

Available games

Quizz

This is a general knowledge quizz in which viewers earn points when they get the right answer. Everyone can propose question on a form accessible on the website. There is 1600+ question available right now ( in french ), divided in categories such as mathematics, history, music etc ... Some voluntary people have a "contributor" role, which gives them a screen when they can see their proposed questions and already existing questions.

Quizz screen

Fairprice

As in the TV show, in this game the viewers have to guess the price of an object displayed on screen. The different objects are stored in a database that I built using a python script which parses Amazon products.

Fairprice screen

Pictionary

In this game, the streamer draws a randomly picked word, the viewers have to guess the word. In order to have a fun game, we used Open Lexicon, on which a list of words with an imageability score is available. After some data filtering ( verb, plural, offense, etc ... ) we got a list of words well suited for pictionary.

In order to do the drawing board I used the simple-drawing-board library on npm, that I modded to add a fill feature. My version is available on npm.

Trader

Trader is a game mocking the trading system, the streamer has to get a certain amount of money, and the viewer can either be with or against him. But most of the time it doesn't matter because some random incongruous events occurs and reverse the course of the game.

In this game, the streamer has a list of word that the twitch chat must guess. In order to do this the streamer tell a word which has a close meaning to those that the chat has to guess. This is inspired by the game "Codenames"

How was it made ?

The website works thanks to 3 parts :

  • The database : A firebase project which contains all the data.
  • The server : A NodeJS server which is the link between the twitch chat and the client.
  • The client : An Angular client, served with nginx.

Firebase Database

This is a classic firebase project, with realtime database used to store data in JSON format and authentication to allow account management on the client. The security of a realtime database is different than a classic SQL Database, as the access are accessible to everyone using the client. I use realtime database access rules which allows me to specify rights on read or write.

NodeJS Server

The NodeJS uses Tmi.JS to communicate with twitch channel, and then uses Socket.io to communicate with the client. It also contains authentication functions.

Angular Client

The Angular client is the biggest part of this application. It can be divided into two parts : the back-office and the games. The games are described up ahead.

I used angular material to create a clean and ergonomic way to manage our content. It includes quizz questions, fairprice products, users and statistics.
